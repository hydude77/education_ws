# Git 101

This file is a simple guide to the words you asked about:

- `git`
- `commit`
- `push`
- `pull`
- GitHub

It also uses this repository, `education_ws`, as the example.

## 1. What is Git?

`Git` is a version control tool.

That means it keeps track of changes to files over time.

You can think of it like:

- a save history for your project
- a change tracker
- a timeline of who changed what and when

If you edit a file today and break it tomorrow, Git helps you see what changed.

## 2. What is a repository?

A `repository` (often called a `repo`) is a project folder that Git is tracking.

In this case, this folder is the repo:

- `/Users/hoyoung/education_ws`

When we ran `git init`, we turned this folder into a Git repository.

## 3. What is GitHub?

`GitHub` is a website that stores Git repositories online.

Git is the tool.
GitHub is a place where Git repositories can live remotely.

That means:

- Git works on your computer
- GitHub stores a copy on the internet
- you can share your code or back it up there

## 4. What is a commit?

A `commit` is a saved checkpoint in your project history.

When you make a commit, you are basically saying:

"I want to record the current changes as one meaningful step."

Example:

- You create `Git101.md`
- You save it with a message like `Add Git101 guide`
- That saved step becomes one commit

Each commit has:

- the changed files
- a unique ID
- a message
- author/time information

## 5. What does `git add` do?

Before a commit, Git lets you choose which changes should be included.

`git add` means:

"Put this change into the staging area so it will be part of the next commit."

Simple flow:

1. Change a file
2. Run `git add <file>`
3. Run `git commit -m "message"`

## 6. What does `git commit` do?

`git commit` creates a snapshot from the staged changes.

Example:

```bash
git add Git101.md
git commit -m "Add Git101 guide"
```

That tells Git:

- include `Git101.md`
- save it in project history
- label that saved point with the message `Add Git101 guide`

## 7. What does `git push` do?

`git push` sends your local commits to the remote repository on GitHub.

Local:

- your computer

Remote:

- GitHub

So:

- `commit` saves the change locally
- `push` uploads that saved change to GitHub

Example:

```bash
git push
```

## 8. What does `git pull` do?

`git pull` brings changes from GitHub down to your computer.

You usually use it when:

- you changed code on another machine
- a teammate pushed changes
- the GitHub repo has commits you do not have yet

Simple idea:

- `push` sends your changes up
- `pull` brings remote changes down

## 9. What does `git status` do?

`git status` shows the current state of your repo.

It helps answer:

- Which files changed?
- Which files are staged?
- Am I ahead or behind GitHub?

This is one of the safest and most useful commands for beginners.

## 11. Basic workflow you can remember

Most of the time, the flow is:

```bash
git status
git add <files>
git commit -m "Short message"
git push
```

If you want to get the latest remote changes first:

```bash
git pull
```

## 12. Helpful plain-English meanings

- `git`: the version control tool
- `repo`: the project Git is tracking
- `commit`: a saved checkpoint
- `push`: upload commits to GitHub
- `pull`: download commits from GitHub
- `status`: show what changed
- `add`: choose what goes into the next commit

## 13. A mental model

Think of it like this:

- your files are your work
- `git add` puts selected work into a box
- `git commit` seals and labels the box
- `git push` ships the box to GitHub
- `git pull` receives boxes from GitHub

## 16. What is a branch?

A `branch` is a separate line of work in the same repository.

You can think of it like making a copy of your project timeline so you can try changes safely.

Common idea:

- `main` is the primary branch
- a new branch is where you try a feature or fix

Example:

```bash
git branch
git checkout -b add-login-page
```

That means:

- show existing branches
- create and switch to a new branch called `add-login-page`

## 17. Why use branches?

Branches are useful because they let you work without disturbing `main`.

That helps when:

- you want to test an idea
- you are building a new feature
- you are fixing a bug
- you want cleaner project history

In team projects, branches are used constantly.

## 18. What is merge?

A `merge` combines changes from one branch into another branch.

Simple example:

1. You create a branch called `add-login-page`
2. You make commits there
3. You go back to `main`
4. You merge that branch into `main`

Example command:

```bash
git merge add-login-page
```

That tells Git:

- take the changes from `add-login-page`
- combine them into the branch I am on now

## 19. What does `git status` really mean?

`git status` is like a dashboard for your repository.

You might see messages like these:

```bash
## main...origin/main
```

This means:

- you are on branch `main`
- your local `main` and GitHub's `origin/main` are in sync

You might also see:

```bash
## main...origin/main [ahead 1]
```

This means:

- you have one local commit that is not on GitHub yet
- you probably need to run `git push`

Or:

```bash
?? notes.txt
```

This means:

- `notes.txt` exists
- Git sees it
- but it is not staged or committed yet

Or:

```bash
M Git101.md
```

This means:

- `Git101.md` was modified
- the change exists locally
- it may or may not be staged depending on the full status output

## 20. A very practical beginner workflow

When you change something, this is a great habit:

```bash
git status
git add <file>
git commit -m "Explain what changed"
git push
```

If you are not sure what is happening, run:

```bash
git status
git log --oneline -5
```

Those two commands answer most beginner questions.

If you remember only this, remember:
